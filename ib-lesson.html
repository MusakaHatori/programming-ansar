// 01 — ОСНОВЫ

Как мыслит хакер

Прежде чем атаковать — нужно понять фундамент. Этичный хакер думает как атакующий, но работает на стороне защиты.

01
РАЗВЕДКА (RECON)
Сбор информации о цели без прямого контакта. OSINT, сканирование DNS, анализ открытых данных. Чем больше знаешь — тем точнее атака.
02
СКАНИРОВАНИЕ
Поиск открытых портов, запущенных сервисов, версий ПО. Каждый открытый порт — потенциальная точка входа. Nmap — главный инструмент.
03
ЭКСПЛУАТАЦИЯ
Использование найденных уязвимостей для получения доступа. SQL-инъекции, XSS, переполнение буфера, социальная инженерия.
04
ПОСТ-ЭКСПЛУАТАЦИЯ
После входа — повышение привилегий, закрепление, сбор данных. В реальном пентесте: документирование и немедленный доклад заказчику.
05
ОТЧЁТНОСТЬ
То чего не показывают в кино. 40% работы пентестера — написание детальных отчётов. Нашёл уязвимость — объясни как её закрыть.
06
CIA ТРИАДА
Confidentiality (конфиденциальность), Integrity (целостность), Availability (доступность). Три столпа ИБ. Любая атака нарушает один или несколько.
// 02 — ПРАКТИКА

Лабораторные работы

5 реальных типов атак. Всё запускается прямо здесь. Никаких реальных систем — только учебная среда.

LAB-01
SQL INJECTION
НАЧАЛЬНЫЙ

// ЧТО ПРОИСХОДИТ

SQL-инъекция — одна из старейших и до сих пор самых частых уязвимостей. Приложение вставляет пользовательский ввод прямо в SQL-запрос без проверки.


Когда ты вводишь логин, код выглядит так:


SELECT * FROM users
WHERE login='ТВОЙ_ВВОД'
AND password='ПАРОЛЬ'

Если ввести специальные символы — можно изменить логику запроса.


// ПОПРОБУЙ

  • Обычный логин: admin
  • Инъекция: admin'--
  • Или: ' OR '1'='1
  • Или: '; DROP TABLE--

// ДЕМО: ПАНЕЛЬ ВХОДА (УЯЗВИМАЯ)

⚠ VULNERABLE LOGIN PANEL v1.0
Подсказка: попробуй логин ' OR '1'='1 с любым паролем
LAB-02
СКАНИРОВАНИЕ ПОРТОВ
НАЧАЛЬНЫЙ

// ЧТО ТАКОЕ ПОРТЫ

Порты — это "двери" компьютера. Каждый сервис слушает на своём порту. Зная какие двери открыты — хакер знает что атаковать.


  • Порт 22 — SSH (удалённый доступ)
  • Порт 80 — HTTP (сайт)
  • Порт 443 — HTTPS (сайт с SSL)
  • Порт 3306 — MySQL (база данных)
  • Порт 21 — FTP (файлы)
  • Порт 3389 — RDP (Windows удалённый рабочий стол)

Открытый порт 3306 напрямую из интернета — это как открытая дверь в хранилище данных.

LAB-03
ВЗЛОМ ХЭШЕЙ ПАРОЛЕЙ
СРЕДНИЙ

// КАК ХРАНЯТСЯ ПАРОЛИ

Пароли никогда не хранятся в открытом виде (в хорошо написанных системах). Хранится хэш — результат необратимой функции.


Пароль: password123
MD5-хэш: 482c811da5d5b4bc6d497ffa98491e38

Атака по словарю: перебираем слова из списка, хэшируем каждое, сравниваем с целевым хэшем.


MD5 — устаревший и небезопасный алгоритм. Современные системы используют bcrypt, Argon2.

// ЦЕЛЬ: ВЗЛОМАТЬ ХЭШ

MD5 HASH TARGET:
5f4dcc3b5aa765d61d8327deb882cf99
LAB-04
МЕЖСАЙТОВЫЙ СКРИПТИНГ (XSS)
СРЕДНИЙ

// ЧТО ТАКОЕ XSS

XSS позволяет внедрить JavaScript-код на страницу так, что он выполняется в браузере другого пользователя.


Представь форму комментариев. Уязвимый сайт выводит ввод пользователя напрямую в HTML без экранирования.


Если написать в комментарий тег <script> — он выполнится у всех кто откроет страницу.


// ЧТО МОЖНО УКРАСТЬ

  • Куки сессии (угон аккаунта)
  • Данные из форм (пароли)
  • Токены авторизации
  • Перенаправить на фишинг

// УЯЗВИМЫЙ ФОРУМ

http://vuln-forum.local/comments
user_alex:
Хороший урок, многое понял
pro_hacker:
SQL инъекции это классика
comment $
Попробуй: <img src=x onerror="alert('XSS!')">
LAB-05
АНАЛИЗ СЕТЕВОГО ТРАФИКА
ПРОДВИНУТЫЙ

// СНИФФИНГ ТРАФИКА

Анализатор пакетов (снифер) перехватывает и анализирует данные которые проходят через сеть. Инструмент: Wireshark.


В незашифрованном HTTP-трафике можно увидеть пароли, куки, данные форм в открытом виде.


В публичных Wi-Fi сетях — любой в той же сети может слушать твой трафик если нет HTTPS.


// НА ЧТО СМОТРЕТЬ

  • HTTP POST — данные форм (пароли)
  • DNS запросы — куда ходит устройство
  • Подозрительные IP — сканирование
  • ARP-запросы — возможная атака MITM

// 03 — РЕАЛЬНОСТЬ

Работа в ИБ:
Red Team vs Blue Team

Две стороны одной медали. Большинство людей хочет в Red Team — и именно туда попасть сложнее всего.

▶ RED TEAM — АТАКУЮЩИЕ
Пентестеры
  • Симулируют реальные атаки на заказчика
  • Ищут уязвимости до того как их найдут реальные хакеры
  • Самая романтичная часть ИБ — и самая редкая
  • Нужно знать: сети, ОС, криптографию, веб, мобильные
  • Зарплата: 250–600к+ (мидл-сеньор)
  • Вакансий: 15–20% от всего рынка ИБ
▶ BLUE TEAM — ЗАЩИТНИКИ
SOC / Аналитики
  • Мониторинг систем, реагирование на инциденты
  • Настройка SIEM, IDS/IPS, файрволов
  • Разбор вредоносов, forensics
  • Писать политики безопасности и обучать сотрудников
  • Зарплата: 150–350к (мидл-сеньор)
  • Вакансий: 80% от всего рынка ИБ
▶ ВХОД В ПРОФЕССИЮ
Путь с нуля
  • Linux и сети — абсолютный фундамент
  • TryHackMe → HackTheBox (бесплатный старт)
  • CTF соревнования — лучшая практика
  • Bug Bounty — реальные деньги без трудоустройства
  • Писать — writeup'ы, статьи, CVE-репорты
  • До первой работы: 18–36 месяцев честно
▶ ОБ ЭТОМ МОЛЧАТ
Тёмная сторона
  • 70% вакансий — compliance и документация
  • Юридическая ответственность огромная
  • Один неверный шаг без разрешения = уголовка
  • Постоянное обучение — не опция, а требование
  • Сертификации стоят $500–2000 за штуку
  • Бизнес не хочет тратить на ИБ — ты продаёшь страх
