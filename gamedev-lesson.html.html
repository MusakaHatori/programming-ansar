// 01 — ОСНОВЫ

ИЗ ЧЕГО СОСТОИТ ИГРА

Любая игра — от Pong до Red Dead Redemption — строится на одних и тех же фундаментальных концепциях.

🔄
GAME LOOP
Сердце любой игры. Бесконечный цикл: принять ввод → обновить состояние → нарисовать кадр. 60 раз в секунду. Остановился — игра зависла.
📦
GAME OBJECT
Каждая сущность в игре — объект. У него есть позиция, скорость, спрайт, коллайдер. Игрок, враг, пуля, платформа — всё это Game Objects.
⚡
DELTA TIME
Время между кадрами. Умножай скорость на delta time — и игра будет одинаково быстрой на любом железе. Без этого — на мощном ПК всё летит.
💥
КОЛЛИЗИИ
Обнаружение столкновений. AABB (прямоугольники), circle (окружности), полигоны. Без коллизий — персонаж проваливается сквозь пол.
🎯
ФИЗИКА
Гравитация, трение, импульс. Реальная физика выглядит скучно — хорошая игровая физика утрирована и ощущается правильно, но законам не подчиняется.
🧠
GAME AI
Конечные автоматы (FSM), поиск пути (A*), поведенческие деревья. Не настоящий ИИ — набор правил "если вижу игрока → атакую".

ФИЗИКА И ПРЫЖОК
ФИЗИКА

// КАК РАБОТАЕТ ПРЫЖОК

Прыжок — это просто отрицательная вертикальная скорость + гравитация которая её уменьшает каждый кадр.

// Каждый кадр: if (onGround && jumpPressed) { vy = -12; // вверх } vy += gravity; // притяжение y += vy * deltaTime;

Меняй гравитацию — меняется ощущение мира. Луна = 0.3, Земля = 0.8, Тяжёлый мир = 2.0


  • Coyote time — прыжок чуть после края платформы
  • Jump buffer — нажал чуть до земли, прыгнет
  • Variable jump height — держишь дольше = выше

// ЧТО ТАКОЕ ЧАСТИЦЫ

Взрывы, огонь, искры, дым, магия — всё это системы частиц. Каждая частица: точка с позицией, скоростью, временем жизни и цветом.

class Particle { constructor(x, y) { this.x = x; this.vx = random(-3, 3); this.vy = random(-8, -2); this.life = 1.0; // 0..1 this.decay = random(0.02,0.05); } }
  • Спавни много — убивай быстро
  • Рандом — основа натуральности
  • Fade out по life значению
  • Object pool — переиспользование

LAB-03
ПОВЕДЕНИЕ ВРАГОВ (FSM)
AI

// КОНЕЧНЫЙ АВТОМАТ

FSM — Finite State Machine. Враг всегда находится в одном из состояний. Условия переключают состояния.

// Состояния врага switch(enemy.state) { case 'patrol': // ходит туда-сюда if(seePlayer()) state='chase'; break; case 'chase': // бежит к игроку if(inRange()) state='attack'; if(!seePlayer()) state='patrol'; break; }
  • Patrol → видит игрока → Chase
  • Chase → в зоне удара → Attack
  • Attack → игрок убежал → Patrol
  • Любое → низкое HP → Flee

// 03 — РЕАЛЬНОСТЬ

ГЕЙМДЕВ:
МЕЧТА vs ПРАВДА

То чего не расскажут в видео "как я стал геймдев разработчиком". Честно.

✅
ЧТО РЕАЛЬНО КРУТО
Видишь продукт в котором люди проводят сотни часов. Технически разнообразно — никогда не скучно. Навыки C++ и математика — ценятся везде. Мощное глобальное комьюнити. Инди — можешь делать что хочешь.
⚠️
CRUNCH КУЛЬТУРА
За 3–6 месяцев до релиза — 60–80 часов в неделю стандарт. Это не "иногда". Это система. CD Projekt Red перед Cyberpunk: ежедневно до 100 часов. Нормализовано до такой степени что люди сами себе не признаются.
💰
ЗАРПЛАТЫ НИЖЕ IT
Разработчик с 5 годами опыта в геймдеве получает на 30–40% меньше чем тот же специалист в корпоративном IT. "Но ведь это игры!" — именно это и позволяет студиям платить меньше. Они продают мечту.
🔥
80% — РУТИНА
Большую часть времени ты не создаёшь крутые механики. Ты чинишь баги в коде 2015 года, оптимизируешь память, делаешь скучные UI-элементы по чужому ТЗ. Интересные задачи — 20% рабочего времени максимум.
😔
ИНДИ РЕАЛЬНОСТЬ
95% инди-игр зарабатывают меньше $1000 за всё время. Средний инди разработчик зарабатывает меньше джуна в банке. Истории успеха Minecraft и Stardew Valley — это лотерея. Выживших ошибка восприятия.
💡
НО ЕСЛИ ГОРИТ — ИДИ
Если ты не можешь не делать игры — иди. Лучший путь: геймдев как хобби + стабильная IT работа → накапливаешь деньги → делаешь инди проект без финансового давления. Многие хиты так и появились.

// 04 — ПУТЬ

КАК ВОЙТИ
В ГЕЙМДЕВ

Реальный путь без романтики. С нуля до первой работы в студии.

1
ФАЗА 01 · 1–3 МЕС
ВЫБЕРИ ДВИЖОК И УЧИСЬ
Unity (C#) — проще старт, больше вакансий в мобилках и инди. Unreal (C++) — AAA студии, сложнее, но ценится больше. Godot (GDScript/C#) — бесплатный, отличный для старта. Пройди официальный туториал полностью.
2
ФАЗА 02 · 3–8 МЕС
СДЕЛАЙ 5 МАЛЕНЬКИХ ИГР
Не одну большую — пять маленьких законченных. Pong, Snake, Arkanoid, платформер, шутер сверху. Законченность важнее масштаба. 90% начинающих бросают одну большую игру на середине.
3
ФАЗА 03 · 8–14 МЕС
GAME JAM — ОБЯЗАТЕЛЬНО
Ludum Dare, Global Game Jam, Brackeys Game Jam. 48–72 часа — сделай игру. Невозможно? Возможно. Это лучшая практика командной работы и дедлайнов. Работодатели любят видеть jam-игры в портфолио.
4
ФАЗА 04 · 14–20 МЕС
ПОРТФОЛИО И GITHUB
Itch.io профиль с реальными играми. GitHub с чистым кодом. Одна "showcase" игра которой не стыдно. Без этого — ты просто человек который "хочет делать игры". С этим — разработчик.
5
ФАЗА 05 · 20–30 МЕС
ПЕРВАЯ РАБОТА
Аутсорс-студии — берут охотнее всего джунов. Стартапы — маленькие команды, учишься быстрее. Первые 2 года — не смотри на зарплату, смотри на скорость роста. Потом деньги придут сами.
